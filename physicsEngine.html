Physics Engine

Project StatusOngoing
Project TypeUniversity Coursework
Software UsedVisual Studio
Languages UsedC++
Primary Role(s)Programmer

This is physics engine submited for a CSC3222 (Gaming Simulations). It has movement based on the Newtonian Dynamics (forces, velocities), collision detection (sphere with sphere, sphere with box, box with box) and physical collision response such as impulse(the object is pushed back, preventing it from exiting the game area).

Its performance is optimised by using broadphase – quad tree nodes, where each node is divided into 4 nodes, if it contains more than 2 objects in it. It continues recursively, until a certain treshold is reached. Physics engine checks for collisions (happens each frame) by seeing if low-level node has more than 1 object within. Then narrowphase checks these objects for a collision. Graphical representation of quad tree nodes are blue rectangles.

Additional optimisation is combining each ground or wall tile into a list of objects – graphically represented by red rectangles. It transforms over a hundred separate objects into several big ones, so the performance is increased drasctically.

It is a first part of the coursework for this module, the second one adds AI, pathfinding and flocking.
