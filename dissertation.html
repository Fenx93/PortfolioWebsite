This is a final year project for Stage 3 at the Newcastle University. The aim is to create a tool to procedurally generate coherent city that considers terrain and does zone planning.

At the moment terrain generation is complete. It uses noise and regions to create a 2d map, that is used when generating vertices for a 3D mesh.

City generation is done via the L-Systems – collection of symbols that recursively generates other collection based on a provided rule. Then this collection is resolved and translated into a required set of roads.

I have edited inspector in unity to be able to run generation without starting the application. Besides saving time it allows me to instantly see the changes applied to the generation algorithms as everything is re-generated automatically when any of the values were changed.

Next step will be the combination of terrain and map and addition of algorithms for this city to be generated in a specific way in a specific place – for example generate as close to the water source and resources as possible.

I am working on it at the moment.
